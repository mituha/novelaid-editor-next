# バックエンド仕様

本文書では、`novelaid-editor-next` のバックエンドの実装仕様について記述します。

## アーキテクチャ概要

本アプリケーションのバックエンドは **Tauri v2** を採用しています。
フロントエンド（React/TypeScript）とバックエンド（Rust）は IPC (Inter-Process Communication) を介して通信します。

主な責務：
- ローカルファイルシステムへのセキュアなアクセス
- ネイティブダイアログの提供
- OS標準機能（ファイル展開など）との連携
- アプリケーション/プロジェクトの設定永続化

## 使用プラグイン

標準的な機能については Tauri 公式プラグインを使用しています。

| プラグイン名 | 用途 | フロントエンドでの利用例 |
| :--- | :--- | :--- |
| `tauri-plugin-fs` | ファイル・ディレクトリ操作 | `readDir`, `readFile`, `writeFile`, `exists`, `mkdir` |
| `tauri-plugin-dialog` | ネイティブダイアログ | フォルダ選択 (`open({ directory: true })`) |
| `tauri-plugin-opener` | 外部連携 | ファイルまたはURLを既定のアプリで開く |

## カスタムコマンド (Rust)

`src-tauri/src/lib.rs` に定義されているカスタムコマンドです。

### `greet`
疎通確認用のプレースホルダーコマンドです。
- **引数**: `name: &str`
- **戻り値**: `String` ("Hello, {name}! ...")

## データ永続化（設定ファイル）

設定ファイルは JSON 形式で保存されます。`ConfigService` (frontend) が `plugin-fs` を介してこれらを管理します。

### アプリケーションレベルの設定
OS標準のアプリ設定ディレクトリ内の `novelaid-next` フォルダに保存されます。
- `config.json`: バージョン情報など
- `settings.json`: テーマ（light/dark/system）、言語設定
- `session.json`: 最後に開いたプロジェクトパス、最近開いたファイル
- `state.json`: ウィンドウサイズ、サイドバーの幅などのUI状態

### プロジェクトレベルの設定
各プロジェクトのルートディレクトリ内の `.novelaid-next` フォルダに保存されます。
- `config.json`: プロジェクト名など
- `settings.json`: プロジェクト固有の設定
- `session.json`: 最後に開いていたファイル、カーソル位置
- `state.json`: ファイルエクスプローラーの展開状態

## ファイルシステム操作

ドキュメント（本文やメモなど）の読み書きは、バックエンドの Rust コードを介さず、フロントエンドから `tauri-plugin-fs` を直接呼び出して行います。

- **読み込み**: `readTextFile` または `readFile`
- **書き込み**: `writeTextFile` または `writeFile`
- **文字コード**: UTF-8 (フロントエンド側で `TextEncoder`/`TextDecoder` を使用)
