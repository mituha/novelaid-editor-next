## フロントエンドとバックエンド

ElectronとTauriはどちらもWEBの技術でデスクトップアプリを構築するフレームワークです。  
そのため、フロントエンドとバックエンドが分離しています。  

フロントエンドはWEBブラウザで表示される部分です。  
今回はTypeScriptとReactを使用しています。  
ブラウザで表示される部分なので、ローカルなファイルシステムにアクセスすることはできません。  
そのため、ファイルシステムにアクセスする必要がある処理はバックエンドに実装します。  

バックエンド部分はElectronとTauriで実装方法が異なります。  
ElectronはNode.jsをバックエンドとして使用します。  
TauriはRustをバックエンドとして使用します。

### チャネル or チャンネル？

今回、AIにバックエンド仕様を書いてもらったわけですが、IPCチャネルの表記がチャンネルとチャネルと混在しています。  
Electron版はチャンネル、Tauri版はチャネルと書かれています。  

Tauriの[公式サイト](https://v2.tauri.app/ja/develop/calling-frontend/
)では、

> チャネル Channels
>《訳注》 チャネル　「流路、伝送路」。日本語では一般に「チャンネル」と表記されますが、コンピュータ用語としては「チャネル」表記が一般的であるので、その表記に従って記述します。

とありました。  
また、Electronの公式サイトでは、チャンネルと表記されています。  

チャンネルは色々な意味で使われる言葉なので、IPCとしてはIPCチャネルと書いとこうかと思います。

### Tauri版でのバックエンド処理の追加

Tauri版では、バックエンド処理の追加はプラグインの範囲で対応されていました。    
便利で良いのですが、ここらでバックエンドとの処理の追加も試してみたいと思います。  

```
今後の拡張を考慮し、IPCチャネルとして、`fs:getDocumentType`を追加して、ファイルのドキュメントタイプの取得をバックエンドで処理するように変更してください。
```

としたんですが、動かねぇーーーっ！　となったんで、色々調査中です。  
ます、`Command fs:getDoccumentType not found`というエラーが出てました。  
これは、ファイル表示がされなくなったので、開発者ツールを起動したら表示されてました。  

ここから、AI君が色々と修正していくのですが、どこが実際に悪かったのかを調べなおす必要があります。  
当初、`fs:`がプラグインの`fs:`と被るので駄目ではということで、コマンド名を色々変更していきました。  
最終的に`get-document-type`というコマンド名になり、ここで別のエラーが発生。  
どうやらコマンドの名称にも色々制約があるようで、更に`permissions/get-document-type.toml`というファイルも作成したりして動作するようにはなりました。  

まずは、この段階での影響範囲を見てみます。

`FileExplorer.tsx`の呼び出し部分。

`lib.rs`にコマンドが追加されています。

```rust
// Learn more about Tauri commands at https://tauri.app/develop/calling-rust/
#[derive(Debug, Clone, Copy, serde::Serialize, serde::Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum DocumentType {
    Novel,
    Markdown,
    Image,
    Chat,
    GitDiff,
    Browser,
    Css,
    Unknown,
}

#[tauri::command(rename_all = "camelCase")]
fn get_document_type(
    path: String,
    is_directory: bool,
    parent_type: Option<DocumentType>,
) -> DocumentType {
    let name = path.to_lowercase();

    if is_directory {
        if name.contains("小説") || name.contains("novel") {
            return DocumentType::Novel;
        }
        if name.contains("設定")
            || name.contains("プロット")
            || name.contains("wiki")
            || name.contains("markdown")
        {
            return DocumentType::Markdown;
        }
        if name.contains("画像") || name.contains("image") {
            return DocumentType::Image;
        }
        return parent_type.unwrap_or(DocumentType::Novel);
    }

    let ext = std::path::Path::new(&path)
        .extension()
        .and_then(|s| s.to_str())
        .map(|s| s.to_lowercase())
        .unwrap_or_default();

    match ext.as_str() {
        "txt" => DocumentType::Novel,
        "md" => DocumentType::Markdown,
        "png" | "jpg" | "jpeg" | "gif" | "webp" | "svg" | "bmp" | "tiff" | "ico" => {
            DocumentType::Image
        }
        "chat" => DocumentType::Chat,
        "css" => DocumentType::Css,
        _ => DocumentType::Unknown,
    }
}

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_dialog::init())
        .plugin(tauri_plugin_fs::init())
        .plugin(tauri_plugin_opener::init())
        .invoke_handler(tauri::generate_handler![get_document_type])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

詳しくは公式ドキュメントの[セキュリティ](https://v2.tauri.app/ja/security/)を参照してください。
とは言うものの、公式ドキュメント読んでもどう書けば良いか良くわからないのですが？！

困った時は初心に戻って公式ドキュメントのコマンドを扱うところを確認します。  
[フロントエンドからRustを呼び出す](https://v2.tauri.app/ja/develop/calling-rust/)のところです。  

基本的に書かれている`my_custom_command`の例と`get_doccument_type`を比較しても違いはなさそうです。  

手動でコード自体をいじって色々確認してみます。

ますは、`default.json`の`permissions`から`allow-get-document-type`を削除。
```
FileExplorer.tsx:58 Failed to read directory: get_document_type not allowed. Permissions associated with this command: allow-get-document-type
```
権限のエラーが出ました。これは現状`permissions`の設定が必要ということ……いや、ちょっとおかしいです。  
エラーの中に`allow-get-document-type`の権限と書かれてますね。
これは、`src-tauri\permissions\get-document-type.toml`の中で追加されている識別子(`identifier`)です。  
恐らくフォルダーにファイルを入れることで自動で組み込まれているのでしょう。
`permissions`フォルダーの下のファイルを消してみます。  

……おや、普通に動作しました。結局、コマンド動作に権限は関係なかった事になります。  
つまり、コマンドの名称の問題だったようです。  

なお、現状フロントエンドからのチャネル名はRust側の関数名が使用され、変更することは出来ないようです。  
issuesに提案としては上がっているっぽいですが、まだ、実装されない段階です。

引数に関しては、`#[tauri::command(rename_all = "camelCase")]`等で変更できます。
プラグインに関してはフロントエンド側でIPCチャネルをラップして隠蔽しているのでTypeScriptの流儀になっています。

そういえば、AI君の最初の実装では`#[tauri::command]`になにやら`fs:getDocumentType`を指定するような引数をつけていました。
つまり、未実装な機能が存在するつもりで動作していないのに気づいていなかったわけです。  
この辺は文法上エラーにならなかったのも問題な気もしますが、AIに任せた修正処理の場合、ドツボにはまることがある良い例でした。
