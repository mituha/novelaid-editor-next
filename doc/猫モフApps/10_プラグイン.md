# 猫モフ Apps - 小説執筆アプリを創ろう - 10. プラグイン

![](./images/20250622_note_1280_670.png)

猫モフ Apps は、猫をモフモフしながら思いついたアイデアを、バイブコーディングでゆるっと創っていく企画です。  

前回はバックエンドの仕様についてまとめました。  
今回は、Tauriのプラグイン機能について見ていきたいと思いもしましたが、面白みに書けるので、プラグインを作成していきます。

なお、プラグインを使う方は`npm run tauri add fs`のように追加するだけです。  
公式サイトの[プラグイン：機能と利用法](https://v2.tauri.app/ja/plugin/)を参考にしてください。

## プラグインの作成

前回、Tauri版にファイルのドキュメントタイプを取得する`get_doccument_type`コマンドを追加しました。  
今回は、このコマンドをプラグインとして作成していきます。  
また、この小説執筆ソフトとしてはファイル操作にドキュメントタイプが絡む場合も多いため、公式のファイルシステムプラグイン(`fs`)を置き換えるようなプラグインを作成する予定でプラグインを作成してみます。  




[プラグインの開発](https://v2.tauri.app/ja/develop/plugins/)

```pwsh
npx @tauri-apps/cli plugin new novelaid-fs
```

実行すると`tauri-plugin-novelaid-fs`というフォルダが作成される。
また、サンプルとしての実行用のプロジェクトとして`examples/tauri-app`が作成されている。
この中で関係しそうな部分を確認。
`package.json`
```json
  "dependencies": {
    "@tauri-apps/api": "^2.0.0",
    "tauri-plugin-novelaid-fs-api": "file:../../"
  },
```
`Cargo.toml`
```toml
[dependencies]
tauri = { version = "2.10.0" }
tauri-plugin-novelaid-fs = { path = "../../../" }
```
`capabilities/default.json`
```json
  "permissions": [
    "core:default",
    "novelaid-fs:default"
  ]
```


```pwsh
npm run tauri add file:./tauri-plugin-novelaid-fs/tauri-plugin-novelaid-fs-api
```
通常のプラグイン追加方法では上手くいかない？

`examples/tauri-app`はサンプルの記載を参考に、追加します。

追加後、`npm i`を実行。  


`build.rs`にコマンドの追加。
```rust
const COMMANDS: &[&str] = &["ping", "get_document_type"];

fn main() {
    tauri_plugin::Builder::new(COMMANDS)
        .android_path("android")
        .ios_path("ios")
        .build();
}
```
これにより、`tauri-plugin-novelaid-fs\permissions\autogenerated\commands\get_document_type.toml`が自動生成される。

`tauri-plugin-novelaid-fs\permissions\default.toml`に以下の権限を追加
```toml
[default]
description = "Default permissions for the plugin"
permissions = ["allow-ping", "allow-get-document-type"]
```
