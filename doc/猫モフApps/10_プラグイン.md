# 猫モフ Apps - 小説執筆アプリを創ろう - 10. プラグイン

![](./images/20250622_note_1280_670.png)

猫モフ Apps は、猫をモフモフしながら思いついたアイデアを、バイブコーディングでゆるっと創っていく企画です。  

前回はバックエンドの仕様についてまとめました。  
今回は、Tauriのプラグイン機能について見ていきたいと思いもしましたが、面白みに書けるので、プラグインを作成していきます。

なお、プラグインを使う方は`npm run tauri add fs`のように追加するだけです。  
公式サイトの[プラグイン：機能と利用法](https://v2.tauri.app/ja/plugin/)を参考にしてください。

## プラグインの作成

前回、Tauri版にファイルのドキュメントタイプを取得する`get_doccument_type`コマンドを追加しました。  
今回は、このコマンドをプラグインとして作成していきます。  
また、この小説執筆ソフトとしてはファイル操作にドキュメントタイプが絡む場合も多いため、公式のファイルシステムプラグイン(`fs`)を置き換えるようなプラグインを作成する予定でプラグインを作成してみます。  

まずは、公式の[プラグインの開発](https://v2.tauri.app/ja/develop/plugins/)を参考に、プラグインを作成していきます。

```pwsh
npx @tauri-apps/cli plugin new novelaid-fs
```
現在のプロジェクトのターミナルで上記コマンドを実行すると`tauri-plugin-novelaid-fs`というフォルダが作成されます。
```
novelaid-editor-next/
├── src/
├── src-tauri/
├── tauri-plugin-novelaid-fs/  ← 作成されたプラグイン
│   ├── examples/
│   │   └── tauri-app/
│   ├── src/
│   ├── Cargo.toml
│   ├── package.json
│   └── ...
├── ...
```

また、サンプルとしての実行用のプロジェクトとして`examples/tauri-app`が作成されています。  
プラグインを単独で作成した場合、このサンプルプロジェクトを実行して動作確認を行うことができるみたいです。  
今回は現在作成中のアプリに組み込むため、サンプルプロジェクト内の関係しそうな部分を確認していきます。
まずは`novelaid-fs`で検索します。名前を変えている場合はその名前を使用してください。

`package.json`
```json
  "dependencies": {
    "@tauri-apps/api": "^2.0.0",
    "tauri-plugin-novelaid-fs-api": "file:../../"
  },
```
フロントエンド(TypeScript)側からの依存関係が設定されています。  
`file:`の形式も使用できるのですね。

`Cargo.toml`
```toml
[dependencies]
tauri = { version = "2.10.0" }
tauri-plugin-novelaid-fs = { path = "../../../" }
```
バックエンド(Rust)側からの依存関係が設定されています。  
こちらも`path`の設定があります。

`capabilities/default.json`
```json
  "permissions": [
    "core:default",
    "novelaid-fs:default"
  ]
```
権限関連っぽい設定もあります。

それでは、開発中のアプリからこのプラグインを使用できるようにしていきます。  
通常のプラグインですと、`npm run tauri add <プラグイン名>`で追加できるのですが、この方法では駄目っぽいです。
```pwsh
npm run tauri add tauri-plugin-novelaid-fs    # そもそもローカルなので見つからない
npm run tauri add ./tauri-plugin-novelaid-fs/ # 駄目っぽい
npm run tauri add file:./tauri-plugin-novelaid-fs/tauri-plugin-novelaid-fs-api # 駄目
```
通常のプラグイン追加方法では上手くいかないみたいなので、`examples/tauri-app`はサンプルの記載を参考に、手動で追加します。
上記の３個所を追加します。パスに関しては階層が違うので注意。

`package.json`
```json
  "dependencies": {
    ...(略)
    "tauri-plugin-novelaid-fs-api": "file:./tauri-plugin-novelaid-fs/"
  },
```

`src-tauri/Cargo.toml`
```toml
[dependencies]
...(略)
tauri-plugin-novelaid-fs = { path = "../tauri-plugin-novelaid-fs/" }
```

`src-tauri/capabilities/default.json`
```json
  "permissions": [
    ...(略)
    "novelaid-fs:default"
  ]
```

追加後、`npm i`を実行すると、`tauri-plugin-novelaid-fs-api`が`node_modules`にインストールされます。
そう言えばパッケージ名としては`tauri-plugin-novelaid-fs-api`が設定されていました。  
微妙に色々な命名規則が絡んでいてややこしいです。  

プラグインのプロジェクトを作成した時点で`ping`コマンドが追加されていました。
これを参考に、`get_document_type`コマンドを追加してください。  
私はAI君に前回追加したコマンドを移植してもらったのですが、権限部分の手順が漏れていて最初動きませんでした。  

どうやら権限関連は手順に則って設定してあれあ自動生成されるようです。

`build.rs`にコマンドの追加。
```rust
const COMMANDS: &[&str] = &["ping", "get_document_type"];

fn main() {
    tauri_plugin::Builder::new(COMMANDS)
        .android_path("android")
        .ios_path("ios")
        .build();
}
```
これにより、`tauri-plugin-novelaid-fs\permissions\autogenerated\commands\get_document_type.toml`が自動生成される。

`tauri-plugin-novelaid-fs\permissions\default.toml`に以下の権限を追加
```toml
[default]
description = "Default permissions for the plugin"
permissions = ["allow-ping", "allow-get-document-type"]
```
